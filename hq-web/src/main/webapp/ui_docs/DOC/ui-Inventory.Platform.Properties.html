Editing a Platform's Network Properties

Topics marked with * relate to HQ Enterprise-only features.

This screen allows users to edit identifying networking information for a platform. All these values are initially auto-discovered and can be edited afterwards. Users can also specify the IP address and port that the HQ Server will use to the connect to the platform's HQ Agent.

Feedback is welcome. Click Add Comment at the bottom of the page.

Tasks Available on This Screen

On this screen, users can do the following tasks:

To change the HQ Server's connection address:port pair on this platform:

  • In Agent Connection, select an IP address:port pair that the HQ Server will use to connect to the HQ Agent, and click .

Note: In HQ Enterprise, a value of -1 for port indicates the channel is for unidirectional communications. For more information see Changing Between Unidirectional and Bidirectional Agent Communications.

There is at a minimum one set of IP Address/Subnet/MAC Address for the loopback (local) IP address — 127.0.0.1 — and then additional sets for each network interface on the device.

To edit a platform's interface information:

  1. To change an already specified set of IP Address, (optional) MAC Address, and (optional) Netmask, replace the existing values in a set.
  2. To delete a set, click Delete this IP, Subnet, and MAC Set under the set to be deleted.
  3. To add a new set, click Add Another Set and then fill in the values.
  4. Click .

The platform's Fully Qualified Domain Name can also be edited. The Platform Type is displayed but cannot be changed.
