Current Health of a Resource

This page presents an overview of the resource's current health, as well as the current health of its related resources. The Current Health page is also a work area where you can compare and analyze the metrics of the resource and its related resources.

Resources Column

In the column are several portlets that display the resource's child resources and/or host resources. Each portlet lists the resources either by name or by type, if there are more than one resource of that type related to the current resource.

For a child resources portlet, HQ displays a two value columns: the Total column denotes how many resources are included; the Avail column displays the last known availability value(s) of the resource(s). Hovering over the icon will bring up the Child Resource menu, which displays:

  • Resource Type
  • Last Known Usage Metric Value
  • Detailed Breakdown of Availability (applies to autogroups only)

For a host resources portlet, HQ displays the Avail column, which shows the last known availability value of the host resource. Hovering over the icon will bring up the Host Resource menu, which displays:

  • Resource Type
  • The Last Known Usage Metric Value

A group or autogroup resource will display a group members portlet. Much like the child or host resources portlet, it displays the list of resources that comprise its group and their current availability. Hovering over the icon will bring up the resource menu, which displays:

  • Resource Type
  • The Last Known Usage Metric Value

The button allows you to include the metrics of the related resources above in the problem metrics portlet below by setting the checkboxes next to any of the other resources and/or types and clicking the button.

At the bottom of this column is the Problem Metrics portlet, which displays the list of metrics that have either collected out-of-bounds values or have caused alerts in the metric display range. You can switch the mode of the portlet to display all metrics by changing the drop-down option to All Metrics. Of course, to switch back to view only the problem metrics, select Problem Metrics in the same drop-down. Each metric is displayed with two counts: O.O.B. and Alerts. The icon menu button appears to the right of the count columns. To display the metric data of the problem metric as a column chart in the workspace on the right-hand side of the page, simply click the icon. Otherwise, hovering over the icon will bring up the context-sensitive problem metrics menu. The menu has the following data and function:

  • Resource Type: the resource type of the metric. This value will reflect the varying resource types when other resources have been selected above. If the resource type represents more than one resource in the hiearchy, then this item will also display the number of resources that have the same problem metric in parenthesis.
  • Problems Began: the time of the first occurrence of O.O.B. metric value or alert
  • Chart Metric in Indicators: click on this option to display a chart for this metric in the Indicators workspace on the right-hand side of the page.
  • View Full Chart: click on this option to navigate to the full chart page of this metric.
  • Metric Data: click on this option to display a popup window which contains the meta data information on this metric and its data.

Indicators Column

The column is basically a work area for you to analyze the health of your resource(s) from a historical perspective. You can aggregate the metric data from the current resource and its related resources together in the charts area and analyze their behavior at any given time.

The View drop-down can be used to save or update the current indicator charts view. In addition, you can navigate to one of your previously saved views by selecting it in the drop-down.

The Availability history displays the resource's availability in the metric display range. Each availability light reflects the average availability over the time slice represented. Clicking on any of the availability lights will bring up or move a highlight bar, which conveniently lines up all of the metrics in the indicator charts with the availability light that you are analysing.

The Indicator Charts are charts of the metric data collected for the specified metric over the metric display range. These charts can be rearranged (with and buttons) or removed (with button) and new charts can be added from the problem metrics portlet. Once you set up a list of metrics in the order that you want, you can save both the metrics and their ordering as a view using the controls above.

Events/Logs Tracking displays markers for time slices that have events associated with them. To view the details of the event(s), click on the blue marker and a details display will appear below the timeline.

The timeline has the same ability to bring up the highlight bar as the availability history. In addition, hovering over any of the time slice will bring up a popup that specifies the time represented by the square, giving you an idea what occurred in that time frame by lining up the availability and the metric data.
